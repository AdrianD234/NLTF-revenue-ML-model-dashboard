# FORECAST_INPUT_TEMPLATE_SPEC.lock.md

## Workbook

The governed forecast input template is generated by `scripts/create_forecast_input_template.py`.

Default output:

`templates/NLTF_forecast_input_template_20q.xlsx`

The default horizon is 20 quarters. A caller may generate a shorter or longer template with:

```powershell
.\.venv\Scripts\python.exe scripts\create_forecast_input_template.py --quarters 1
.\.venv\Scripts\python.exe scripts\create_forecast_input_template.py --end-period 2050Q4
```

Template filenames include the requested horizon:

- `NLTF_forecast_input_template_1q.xlsx`
- `NLTF_forecast_input_template_20q.xlsx`
- `NLTF_forecast_input_template_to_2050Q4.xlsx`

`--quarters` must be at least 1. `--quarters` and `--end-period` are mutually exclusive.

## Sheet Contract

The workbook must contain exactly these sheets:

- `README`
- `PED Inputs`
- `Light RUC Inputs`
- `Heavy RUC Inputs`

Each input sheet contains future quarters beginning with the quarter after the latest known actual in `data/dashboard_evidence_pack/data/scorecard_predictions.parquet`.

For the current evidence pack, the latest known actual is `2025Q4`, so the default committed template covers `2026Q1` through `2030Q4`.

## Period And Horizon Rows

- Generated rows include `period`, `year`, `quarter` and `horizon`.
- Formula, dummy, log, difference and interaction columns are generated for every template row.
- Users may fill 1 quarter, the full default 20 quarters, or any continuous prefix through a chosen end period.
- Blank trailing rows are allowed and ignored by the runner.
- Non-blank valid rows must be continuous from the first generated forecast quarter.
- PED, Light RUC and Heavy RUC sheets must have the same continuous valid periods.

## Protection And Entry Rules

- `period`, `year`, `quarter`, `horizon` are generated system columns.
- User-entry columns are highlighted and unlocked.
- Formula columns are protected.
- Every header has a comment explaining its role and unit.
- Users fill only user-entry columns.
- Required user-entry values must be positive numbers.

## Required Header Families

The input sheets must include:

- quarter/year/horizon identifiers;
- GDP or GDP per capita fields where required;
- population where required;
- unemployment where required;
- real petrol, diesel, Light RUC or Heavy RUC prices where required;
- Q2/Q3/Q4 dummies;
- post-2020 dummy;
- trend/time index;
- target lag/history values where required;
- derived log, lag, difference and interaction columns.

## Transform Rule

The Excel formulas help users inspect the workbook, but the Python runner recomputes transforms from user-entry values during validation and pack creation. The committed formulas are therefore a user-audit aid, not the source of truth for model scoring.
